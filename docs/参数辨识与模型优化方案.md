> **适用项目**: robotruck_school_1128  
> **文档日期**: 2026-06-24  
> **状态**: 方案设计阶段  
> **作者**: 宋云峰 (yunfeng.song)

---

## 背景

针对从纯仿真环境向真实重型设备（如 150 吨矿用卡车）工程落地的需求，为消除 **"Sim-to-Real" 的域偏差（Domain Gap）**，本方案规划了利用历史实车运行数据对底层控制与状态估计算法进行数据驱动优化的实施路径。

本方案可为系统中的 **MPC 预测模型**和各类**卡尔曼观测器（EKF/UKF）**提供精准的物理参数、最优的噪声协方差矩阵以及非线性误差补偿，便于团队评估并在多项目中复用。

## 阶段一：基于历史数据的物理参数灰盒辨识

**Grey-box Identification**

### 核心目标

从实车历史数据中提取真实的车辆物理参数与执行器特性，为状态估计节点（如 `veh_param_est`）提供高精度的初始值及边界约束，避免实车部署初期的滤波器发散。

### 实施方法

构建离线的车辆运动学/动力学物理模型。将历史数据集中的控制指令与状态输入作为模型激励，令模型输出相应的预测状态。

**模型激励（输入）**：
- 前轮转角 $\delta$
- 车速 $v_x$

**模型预测输出**：
- 侧向加速度 $a_y$
- 横摆角速度 $\dot{\gamma}$

### 优化目标

设立损失函数（Loss Function），最小化模型预测输出与实际底盘和高精度惯导传感器测量值之间的均方误差（MSE）。

$$\mathcal{L} = \frac{1}{N} \sum_{i=1}^{N} \left[ (a_y^{\text{pred}} - a_y^{\text{meas}})^2 + (\dot{\gamma}^{\text{pred}} - \dot{\gamma}^{\text{meas}})^2 \right]$$

### 算法选型

| 方案 | 适用场景 | 优势 |
|------|----------|------|
| **Levenberg-Marquardt** | 参数较少（3-5个）、模型解析 | 收敛快，无需梯度计算框架 |
| **PyTorch 自动求导** | 参数较多、含延迟建模 | 微分方程计算图，灵活扩展 |

### 待辨识参数清单

| 参数 | 符号 | 当前初值（空载） | 单位 |
|------|------|-----------------|------|
| 前轴侧偏刚度 | $C_f$ | 1.4×10⁵ | N/rad |
| 中轴侧偏刚度 | $C_m$ | 1.2×10⁵ | N/rad |
| 后轴侧偏刚度 | $C_r$ | 1.2×10⁵ | N/rad |
| 整车绕Z轴转动惯量 | $I_z$ | 6.5×10⁵ | kg·m² |
| 系统纯延迟时间 | $\tau$ | — | s |

## 阶段二：卡尔曼滤波协方差矩阵的黑盒调优

**Black-box Optimization**

### 核心目标

解决 EKF/UKF 中过程噪声协方差矩阵 $Q$ 和测量噪声协方差矩阵 $R$ 人工调参耗时且难以达到全局最优的问题。

### 实施方法

对历史实车运行数据进行特征切片，按矿区典型作业场景建立不同工况的测试数据集：

| 工况标签 | 描述 | 关键动态 |
|----------|------|----------|
| `loaded_uphill` | 满载上坡 | 大扭矩、高滑移率 |
| `empty_downhill` | 空载下坡 | 制动能量、低附着 |
| `muddy_turn` | 泥泞转弯 | 大侧偏角、低μ |
| `flat_straight` | 平路直行 | 稳态基准 |

### 算法选型

**超参数定义**：将 $Q$ 矩阵和 $R$ 矩阵的对角线元素定义为待优化的超参数。

$$\theta = \{q_1, q_2, ..., q_n, r_1, r_2, ..., r_m\}$$

### 优化引擎

参照以下两种算法，根据参数空间维度选择：

| 算法 | 适用维度 | 特点 |
|------|----------|------|
| **贝叶斯优化 (Bayesian Optimization)** | ≤20维 | 采样效率高，适合昂贵评估 |
| **遗传算法 (Genetic Algorithm)** | >20维 | 全局搜索，并行评估 |

### 评估流程

1. 在离线仿真节点中回放切片数据
2. 运行 EKF/UKF 算法
3. 计算估计结果与高精度 RTK 后处理真值（Ground Truth）之间的残差
4. 优化引擎在高维空间中自动启发式搜索，输出使全局残差最小的 $(Q, R)$ 参数组合

## 阶段三：基于 PINN 架构的残差补偿机制

**Residual Compensation**

### 核心目标

弥补传统 8-DOF 动力学解析模型在表达重载车辆高度非线性特征时的拟合缺陷，进一步提升 MPC 预测模型的精度。

**模型无法覆盖的非线性因素**：
- 空气悬架非线性侧倾
- 轮胎极限磨损
- 重载车架扭转变形

### 双驱动架构

采用 **"物理机理解析解 + 数据驱动误差补偿"** 的混合架构（类 PINN 思路）：

```
┌──────────────┐     ┌──────────────┐
│  8-DOF 物理   │     │  神经网络     │
│  解析模型     │     │  (MLP/LSTM)  │
│              │     │              │
│  输入: δ,vx  │     │  输入: 综合   │
│  输出: 标称   │     │  观测状态    │
│  预测状态     │     │  输出: 残差   │
└──────┬───────┘     └──────┬───────┘
       │                    │
       └────────┬───────────┘
                ▼
    最终预测 = 物理计算值 + 神经网络补偿项
```

### 神经网络设计

| 属性 | 建议 |
|------|------|
| **架构** | 小型 MLP（3-5层）或 LSTM（序列建模） |
| **输入** | 车速、载荷、转向角、加速度、滑移率 |
| **输出** | 物理模型预测值与真实传感器观测值之间的动态误差（Residual） |
| **训练** | 离线，历史数据监督学习 |

### 最终预测状态

$$x_{\text{final}} = x_{\text{physics}}(\delta, v_x; \theta) + \Delta x_{\text{NN}}(v, m, \delta, a, \lambda)$$

此架构确保：
- **极端未知工况**：物理模型安全兜底（保障稳定性）
- **常态和复杂工况**：神经网络带来极高拟合精度

## 实施注意事项与数据要求

### 数据激励性要求 (Sufficient Excitation)

为使上述数据驱动方案取得预期效果，对所调用的历史数据集有极高的数据激励性要求：

1. **避免数据同质化**
   - 若历史数据全为低速、平缓直行工况，模型将无法学习到轮胎的侧偏刚度与非线性特性
   - 容易产生严重过拟合

2. **工况覆盖度**
   需要筛选出包含相对激烈动态响应的数据片段，确保轮胎和悬架产生了足够的侧向力与纵向滑移率：

   | 数据特征 | 最低要求 |
   |----------|----------|
   | 较快速度过弯 | 侧向加速度 > 0.1g |
   | 急加速/急减速 | 纵向加速度 > 0.15g |
   | 载荷剧烈变化 | 空载→满载切换片段 |
   | 多路面附着 | μ 覆盖 0.3-0.9 |

3. **数据量评估**
   - 灰盒辨识：≥ 30 分钟有效动态数据
   - 协方差调优：≥ 5 个完整工况循环，每种 ≥ 3 分钟
   - PINN 训练：≥ 2 小时多工况混合数据
